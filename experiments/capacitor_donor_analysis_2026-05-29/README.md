## Initial findings

Capacitor is CDB-backed.

The empty base has:

```text
ROOT.CDB size: 118 bytes
```

The single-capacitor donors have:

```text
ROOT.CDB size: 252 bytes
```

The resistor-capacitor series donor has:

```text
ROOT.CDB size: 350 bytes
```

So capacitor cannot be treated as a DSN-only terminal-like object. It needs both:

```text
ROOT.CDB component record
ROOT.DSN visual object record
```

## ROOT.DSN findings

The capacitor visual data includes these marker families:

```text
COMPONENT ID
COMPONENT VALUE
SUBCKT NAME
PROPERTIES
CAPACITOR
CAP
CAP10
```

The capacitor object is different from a resistor object and must not be generated by only replacing text markers.

## Important warning

Do not derive capacitor by only changing a resistor marker to `CAPACITOR`.

Capacitor requires a correct CDB record and a correct DSN visual record.
